<objective>
## Goal
The groove engine produces audible, measurable micro-timing character on BEAT and BASS.
DRAG makes hits late, PUSH makes them early, ROLL multiplies a step, STUTTER gates
repetitions, FRACTURE fires probabilistically. MonoSynth gains an ADSR envelope.

## Purpose
F3-AC1 through F3-AC5 all require a working AestheticResolver. F3-AC4 (STUTTER)
explicitly covers BASS. MonoSynth ADSR gives the BASS note shape needed for STUTTER
to be audible (gated repetitions need a quick release, not an indefinitely sustaining tone).

## Output
- `Source/audio/AestheticResolver.h/.cpp` — applies all 5 aesthetics; returns
  transformed event list into a pre-allocated output buffer (no allocation)
- `Source/model/Schema.h` — Aesthetics extended with all groove parameters
- `Source/audio/BeatSequencer.h/.cpp` — uses AestheticResolver + carry-forward queue
- `Source/audio/MonoSynth.h/.cpp` — manual ADSR envelope added
</objective>

    **AestheticResolver.h:**
    ```cpp
    struct ResolvedEvent {
        int  midi_note;       // -1 for BEAT drum
        float velocity;
        int  sample_offset;   // absolute offset within block; may be deferred
        bool deferred;        // true = belongs in next block (carry-forward)
        std::string voice_type_or_layer; // for routing
        float param1;
    };

    class AestheticResolver {
    public:
        void prepare(double sampleRate, int maxBlockSize) noexcept;

        // Audio thread: transform one event (from BEAT DrumTrack or BASS Layer.events)
        // applying aesthetics. Appends results to outputBuf[]. Returns count added.
        // Uses carry-forward queue for events displaced outside [0, blockSize).
        int resolveEvent(const Event& event, const Aesthetics& aes,
                         float baseVelocity, float param1,
                         int beatSampleOffset, int blockSize,
                         ResolvedEvent* outputBuf, int maxOut) noexcept;

        // Drain carry-forward events from previous block (call at START of each block)
        int drainPending(ResolvedEvent* outputBuf, int maxOut) noexcept;

        // Reset fracture LCG to seed (call at pattern loop start: beat_index % 16 == 0)
        void resetFractureSeed(uint32_t seed) noexcept { lcgState_ = seed; }

    private:
        double sampleRate_  = 44100.0;
        double samplesPerStep_ = 0.0; // set from BPM and step count in prepare

        // Carry-forward queue (Strategy B from F3.99 spike)
        static constexpr int kQueueCap = 8;
        ResolvedEvent pendingQueue_[kQueueCap];
        int pendingCount_ = 0;

        uint32_t lcgState_ = 12345u;
        uint32_t nextLCG() noexcept {
            lcgState_ = lcgState_ * 1664525u + 1013904223u;
            return lcgState_;
        }
    };
    ```

    **AestheticResolver::resolveEvent() implementation:**

    1. **FRACTURE check first** (before any other aesthetic):
       If aes.fracture_prob > 0: roll LCG, if `(nextLCG() % 1000) / 1000.0f > fracture_prob` → return 0 (event dropped).
       At loop start (beat_index % 16 == 0): resetFractureSeed(aes.fracture_seed) called by BeatSequencer.

    2. **Compute base sample offset** with DRAG/PUSH:
       `samplesPerStep_ = sampleRate_ * 60.0 / bpm / stepsPerBeat` — set from context.
       Instead: AestheticResolver doesn't know BPM directly. Receive `stepPeriodSamples`
       as a parameter (computed by BeatSequencer from Clock's samplesPerBeat_).
       ```
       drag_samples = static_cast<int>(aes.drag * stepPeriodSamples * 0.5);  // max 50% of step
       push_samples = static_cast<int>(aes.push * stepPeriodSamples * 0.3);  // max 30% of step
       offset = beatSampleOffset + drag_samples - push_samples;
       ```
       If offset < 0: add to pendingQueue_ with adjusted offset (offset + blockSize).
       If offset >= blockSize: add to pendingQueue_ with offset - blockSize.
       Otherwise: emit at offset.

    3. **ROLL** (applied to the (possibly displaced) base event):
       If aes.roll_mult > 1 AND offset is in [0, blockSize):
       Compute spacing = stepPeriodSamples / roll_mult. Emit roll_mult events:
       offsets[k] = offset + k * spacing (clamped/deferred per above).
       Velocities: baseVelocity * pow(aes.roll_vel_decay, k).

    4. **STUTTER** (applied to base event):
       If aes.stutter_reps > 0:
       gate_samples = static_cast<int>(aes.stutter_gate * stepPeriodSamples).
       Emit stutter_reps events starting at offset, spaced by gate_samples.
       <!-- audit-added M4: MonoSynth.trigger() resets phase_=0.0 causing click on retrigger.
            For STUTTER, use a retrigger-without-phase-reset mode to avoid click artifacts. -->
       **MonoSynth phase continuity for STUTTER**: add a `triggerRetain(velocity)` method
       that resets the envelope (envPos_=0, envVal_=0.0f) but does NOT reset phase_.
       This preserves waveform continuity across stutter repetitions (no click at retrigger).
       `trigger()` keeps resetting phase_ (used for new notes where clean attack is desired).
       `triggerRetain()` is used when STUTTER emits sub-events within the same step.
       BeatSequencer: use `triggerRetain` for STUTTER sub-events 2..N; use `trigger` for event 1.

    <!-- audit-added M2: Clock must expose samplesPerBeat() for BeatSequencer to compute stepPeriodSamples -->
    **Clock.h**: add accessor `double samplesPerBeat() const noexcept { return samplesPerBeat_; }`
    This makes `samplesPerBeat_` readable from BeatSequencer without making the field public.

    **BeatSequencer.process() extensions:**
    1. At start: call `resolver_.drainPending(pendingBuf, maxOut)` → dispatch deferred events from last block
    <!-- audit-added M3: FRACTURE seed reset MUST happen AFTER drain, not before -->
    2. On pattern loop start (beat_index % pattern.length_steps == 0): call seed reset
       ONLY AFTER drainPending() has dispatched all pending events from the PREVIOUS bar.
       Order must be: drainPending() → resetFractureSeed() — never the reverse.
       Resetting before drain would alter the LCG state mid-sequence, producing
       non-deterministic results for events that span bar boundaries.
    3. For each BEAT drum event: call `resolver_.resolveEvent(event, track.aesthetics, ..., stepPeriodSamples, blockSize, ...)`
    4. For each BASS event: same, using layer.aesthetics
    5. BeatSequencer computes `stepPeriodSamples = samplesPerBeat_` from Clock (pass as parameter or get from snap.tempo + sampleRate)

    Add `double stepPeriodSamples` parameter to BeatSequencer::process().
    PluginProcessor passes this as `clock_.samplesPerBeat()` — add this accessor to Clock.

    **CMakeLists.txt**: add AestheticResolver.cpp.

    **Add to MonoSynth private members:**
    ```cpp
    int   attackSamples_   = 0;
    int   decaySamples_    = 0;
    float sustain_         = 0.8f;
    int   releaseSamples_  = 0;
    int   envPos_          = 0;
    float envVal_          = 0.0f;
    bool  releasing_       = false;
    float releaseFrom_     = 0.0f;
    ```

    **trigger()**: compute envelope durations from params:
    ```cpp
    attackSamples_  = juce::jmax(1, (int)(params_.env_attack_ms  / 1000.0f * sampleRate_));
    decaySamples_   = juce::jmax(1, (int)(params_.env_decay_ms   / 1000.0f * sampleRate_));
    sustain_        = params_.env_sustain;
    releaseSamples_ = juce::jmax(1, (int)(params_.env_release_ms / 1000.0f * sampleRate_));
    envPos_ = 0; envVal_ = 0.0f; releasing_ = false;
    active_ = true;
    ```

    **release()**: set releasing_=true; releaseFrom_=envVal_; envPos_=0.

    **process() inner loop**: before writing output, compute envVal_:
    - If !releasing_: same manual ADSR ramp as in the earlier failed attempt,
      but now applied AFTER the oscillator sample:
      ```
      if      (envPos_ < attackSamples_)                    envVal_ = (float)envPos_/attackSamples_;
      else if (envPos_ < attackSamples_ + decaySamples_)    envVal_ = 1.0f - (float)(envPos_-attackSamples_)/decaySamples_ * (1.0f-sustain_);
      else                                                   envVal_ = sustain_;
      ++envPos_;
      ```
    - If releasing_: envVal_ = releaseFrom_ * (1.0f - (float)envPos_/releaseSamples_); ++envPos_;
      When envPos_ >= releaseSamples_: envVal_=0; active_=false.

    Output: `ch[s] += osc * envVal_ * velocity_ * 0.4f` (same as current, just multiply by envVal_).

    Use BassVoiceParams fields for envelope timing:
    - env_attack_ms  = 5ms default
    - env_decay_ms   = 100ms default
    - env_sustain    = 0.8 default
    - env_release_ms = 80ms default

    These are already in BassVoiceParams from F2 Schema. MonoSynth::trigger() receives
    BassVoiceParams — use them directly. No new fields needed.
