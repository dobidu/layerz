<objective>
## Goal
LAYERZ supports multiple patterns per project, a playback chain (A→B→A→B), pattern
switching at bar boundaries, and save/load of .layerz files via keyboard shortcuts.

## Purpose
F2-AC2 (pattern switching), F2-AC3 (chain playback), and F2-AC4 (save/load) all depend
on this plan. Without multi-pattern support, LAYERZ is locked to a single 16-step loop.

## Output
- `Source/audio/ChainManager.h/.cpp` — tracks active pattern, advances chain at bar boundary
- `Source/gui/PatternSelector.h/.cpp` — A/B/+ pattern buttons wired to ProjectStore
- `Source/PluginEditor.h/.cpp` — save/load buttons in top bar
- Updated BeatSequencer: reads active pattern index not hardcoded patterns[0]
- Updated PluginProcessor: integrates ChainManager per processBlock
</objective>

## Key constraints
- Pattern switching MUST happen at a bar boundary — never mid-pattern (audio glitch risk)
- No allocations in ChainManager::process() on audio thread
- Project.active_pattern_index is the "edit view" index; ChainManager tracks playback index
- Save file dialog is a GUI-thread operation; must not block audio thread
</context>

  <action>
    ```cpp
    class ChainManager {
    public:
        void prepare(double sampleRate) noexcept;

        // Audio thread: called once per processBlock with current beat events.
        // Returns the pattern index to use for THIS block.
        // Advances chain at bar boundaries (beat_index % stepsPerBar == 0).
        // Pre-reads chain from snapshot — no allocation, no lock.
        int process(const BeatEvents& beatEvents,
                    const Project& snap) noexcept;

        // GUI thread: request an immediate pattern switch at next bar boundary.
        void requestPatternSwitch(int patternIndex) noexcept;

        int currentPatternIndex() const noexcept {
            return playIndex_.load(std::memory_order_relaxed);
        }

    private:
        double sampleRate_    = 44100.0;
        std::atomic<int> playIndex_      { 0 };
        std::atomic<int> pendingIndex_   { -1 }; // -1 = no pending switch
        int chainPosition_    = 0;  // current position in chain array
        int barsRemaining_    = 1;  // bars left in current chain entry
        bool chainActive_     = false;
    };
    ```

    **process() logic:**
    1. Read chain from snap.chain (immutable for this block)
    <!-- audit-added M4: guard against empty chain BEFORE any modulo operation to prevent crash -->
    2. If snap.chain.empty(): return juce::jlimit(0, (int)snap.patterns.size()-1,
       playIndex_.load(std::memory_order_relaxed));  // bounds-safe return, no chain active
    3. For each BeatEvent: check if beat_index % length_steps == 0 (bar boundary at step 0)
       - Use: beat_index % 16 == 0 as approximation (all patterns 16-step in F2)
       - Note: this assumes all patterns are 16 steps. Add comment in code.
    4. At bar boundary:
       - Decrement barsRemaining_
       - If barsRemaining_ <= 0:
         <!-- audit-added M4: re-check chain not empty before modulo (could change between guard and here) -->
         - if (snap.chain.empty()) { barsRemaining_ = 1; return playIndex_.load(); }
         - Advance chainPosition_ = (chainPosition_ + 1) % snap.chain.size()
         - Find pattern index for snap.chain[chainPosition_].pattern_id
         - Set barsRemaining_ = snap.chain[chainPosition_].bars
         - playIndex_.store(newIdx, memory_order_relaxed)
    5. Also check pendingIndex_: if >= 0 AND bar boundary:
       - playIndex_.store(pendingIndex_, memory_order_relaxed)
       - pendingIndex_.store(-1, memory_order_relaxed)
       - Reset chainPosition_ to match the manually-selected pattern
    6. Return playIndex_.load()

    **Helper**: find pattern index by ID:
    ```cpp
    static int findPatternIndex(const Project& snap, const std::string& id) noexcept {
        for (int i = 0; i < (int)snap.patterns.size(); ++i)
            if (snap.patterns[i].id == id) return i;
        return 0;
    }
    ```

    Zero heap allocations in process() — reads snap by const ref, uses only atomics + int fields.
