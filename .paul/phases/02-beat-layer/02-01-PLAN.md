<objective>
## Goal
LAYERZ produces audible drum hits (kick, snare, hat, perc) from a BEAT layer's step
pattern. Audio thread reads events from ProjectStore snapshot and triggers voices at
the correct sample offsets. Debug builds enforce RT safety.

## Purpose
F1-AC1 (audible pattern), F1-AC4 (RT safety) depend entirely on this plan.
The UI in 02-02 is cosmetic without a working audio engine beneath it.
This plan replaces the F0 click harness with real drum synthesis.

## Output
- `Source/model/Schema.h` — DrumTrack struct; Layer extended with drum_tracks[]
- `Source/audio/DrumVoice.h/.cpp` — kick, snare, hat, perc synthesis (JUCE DSP)
- `Source/audio/VoiceBank.h/.cpp` — manages 4 voice instances
- `Source/audio/BeatSequencer.h/.cpp` — reads Layer.drum_tracks[], dispatches triggers
- `Source/audio/AudioThreadGuard.h` — thread-local RT safety flag (debug builds)
- `Source/PluginProcessor.h/.cpp` — wired to BeatSequencer; test pattern hardcoded

Note: Faust deferred for drum voices. JUCE DSP (oscillators, noise, filters, envelopes)
covers kick/snare/hat/perc at F1 complexity. Faust enters in F4 (BLOOM) and F5 (PULVERIZE).

    **DrumVoice.h/.cpp** — one class, voice_type selects synthesis path:

    ```cpp
    class DrumVoice {
    public:
        void prepare(const juce::dsp::ProcessSpec& spec) noexcept;
        // Called from audio thread — no allocation, no lock
        void trigger(float velocity, float param1) noexcept;
        // Adds rendered voice output into buf[channel][sample..sample+numSamples]
        void process(juce::AudioBuffer<float>& buf, int startSample, int numSamples) noexcept;
        void setType(const std::string& type) noexcept;
        bool isActive() const noexcept { return envelope_pos_ < envelope_len_; }
    private:
        // ... voice state: phase, envelope, filter state, etc.
    };
    ```

    Synthesis per type (all using JUCE DSP primitives, no allocations after prepare()):

    **kick**: sine oscillator, exponential pitch envelope (param1 = pitch_decay_ms, range 20–500ms).
    pitch_hz starts at 180Hz, decays to 50Hz over param1. Amplitude envelope: instant attack,
    exponential decay ~400ms. Write: buf[0][s] += amplitude * sin(2π * phase).
    phase advances by pitch_hz(t) / sampleRate per sample.

    **snare**: white noise (LCG rand, not std::rand — deterministic, no stdlib call overhead)
    multiplied by an exponential envelope (param1 = decay_ms, 30–200ms).
    Add a tuned sine at ~220Hz (tone component) at 30% level.

    **hat**: white noise → 2-pole highpass filter (cutoff ~8000Hz, Q=0.7, juce::dsp::IIR).
    Envelope: instant attack, very short exponential decay (param1 = decay_ms, 10–80ms).
    Prepare the filter coefficients in prepare() — never reallocate in process().

    **perc**: sine burst at param1 Hz (60–800Hz range), short fixed envelope (50ms).

    All voices: pre-allocate any per-voice buffers in prepare(). ZERO allocations in trigger() or process().

    <!-- audit-added M1: retrigger behavior must be explicit -->
    **Retrigger behavior**: trigger() RESETS the voice envelope from the beginning regardless of whether
    the voice is already sounding. This is correct drum machine behavior (kick retriggers at high BPM).
    Do NOT ignore a trigger while active; do NOT blend — reset and restart.

    <!-- audit-added M2: buffer bounds clamping required to prevent overflow -->
    **process() bounds clamping**: DrumVoice::process() MUST clamp writes to the buffer boundary:
    ```cpp
    int avail = buf.getNumSamples() - startSample;
    int write_count = juce::jmin(remaining_envelope_samples, avail);
    // write only write_count samples starting at startSample
    ```
    A trigger near end-of-block (high sampleOffset) leaves fewer samples than the full envelope.
    Writing past buf.getNumSamples() is undefined behaviour.

    <!-- audit-added SR2: deterministic but randomized LCG seed -->
    **LCG noise seed**: Initialize the LCG seed in prepare() using a value derived from
    `static_cast<uint32_t>(std::random_device{}())`. This prevents the snare/hat from
    producing the identical noise sequence every time LAYERZ starts.

    **VoiceBank.h/.cpp:**
    ```cpp
    class VoiceBank {
    public:
        void prepare(double sampleRate, int maxBlockSize) noexcept;
        // Audio thread — no alloc, no lock
        void trigger(const std::string& voice_type, float velocity, float param1,
                     int sampleOffset, juce::AudioBuffer<float>& buf) noexcept;
        void setLevel(const std::string& type, float level) noexcept;
        void setMute(const std::string& type, bool muted) noexcept;
    private:
        DrumVoice voices_[4];
        float     levels_[4]  = {1,1,1,1};
        bool      mutes_[4]   = {false,false,false,false};
        static int typeIndex(const std::string& t) noexcept; // kick=0,snare=1,hat=2,perc=3
    };
    ```

    trigger() writes directly into the output buffer at sampleOffset.
    Level is applied as a gain on the voice output (multiply sample values).
    Mute check is before triggering: if muted, early return without writing.
    No heap allocation anywhere in the process path.

    **AudioThreadGuard.h** (header-only):
    ```cpp
    struct AudioThreadGuard {
        AudioThreadGuard()  noexcept { active() = true; }
        ~AudioThreadGuard() noexcept { active() = false; }
        static bool& active() noexcept {
            static thread_local bool flag = false;
            return flag;
        }
    };
    // In debug builds, jassert checks can use AudioThreadGuard::active()
    // to detect audio-thread violations in any code path.
    #define LAYERZ_ASSERT_NOT_AUDIO_THREAD() \
        jassert(!AudioThreadGuard::active())
    ```

    **BeatSequencer.h/.cpp:**
    ```cpp
    class BeatSequencer {
    public:
        void prepare(double sampleRate) noexcept;

        // Audio thread: for each beat event, scan drum_tracks for matching step,
        // trigger the voice via bank. No allocations.
        void process(const juce::AudioPlayHead::PositionInfo& pos,
                     const Project& snap,
                     const BeatEvents& beatEvents,
                     juce::AudioBuffer<float>& buf,
                     VoiceBank& bank) noexcept;
    };
    ```

    process() implementation:
    - Guard against empty patterns: if snap.patterns.empty(), return.
    - Use snap.patterns[0] for now (multi-pattern support is F2).
    - For each BeatEvent in beatEvents:
      - beat_step = beat_event.beat_index % pattern.length_steps
      - For each DrumTrack in layer.drum_tracks:
        - Search track.events for an Event with event.step == beat_step
        - If found AND !track.mute:
          bank.trigger(track.voice_type, event.velocity, track.param1,
                       beat_event.sample_offset, buf)
    - No allocation in this loop. events[] is small (≤16 entries); linear scan is fine.

    **PluginProcessor changes:**
    - Add `VoiceBank voiceBank_` and `BeatSequencer sequencer_` members
    - Remove `clickBuffer_[]` and click-writing code
    - `prepareToPlay()`: call `voiceBank_.prepare(sampleRate, blockSize)` and `sequencer_.prepare(sampleRate)`
    - `processBlock()`:
      1. `AudioThreadGuard guard;` — at top, sets thread-local flag
      2. `auto snap = store_.snapshot();` — short mutex
      3. Get PositionInfo, call `clock_.process(pos, numSamples)` → beatEvents
      4. Call `sequencer_.process(pos, *snap, beatEvents, buffer, voiceBank_)`
      5. Zero-out channel 1 if present (mono voices write to channel 0 only for now)

    <!-- audit-added M1: C++20 required for lock-free snapshot — F0.99b spike promised this upgrade in F1 -->
    **CMakeLists.txt C++20 upgrade (MUST-HAVE for F1-AC4)**:
    Change `set(CMAKE_CXX_STANDARD 17)` → `set(CMAKE_CXX_STANDARD 20)`.
    JUCE 8 supports C++20. AppleClang 15+, GCC 11+, MSVC 2022 all support C++20.
    CI will verify on all three platforms.

    **ProjectStore upgrade to lock-free snapshot (MUST-HAVE for F1-AC4)**:
    Replace the C++17 mutex-based `current_` with C++20 `std::atomic<std::shared_ptr<Project>>`:
    ```cpp
    // ProjectStore.h: replace mutable std::mutex + std::shared_ptr<Project> with:
    std::atomic<std::shared_ptr<Project>> current_;
    std::mutex mutation_mutex_; // serializes GUI-thread mutations only

    // snapshot() — NO mutex on audio thread (truly lock-free):
    std::shared_ptr<const Project> snapshot() const noexcept {
        return current_.load(std::memory_order_acquire);
    }

    // postMutation() — mutex on GUI thread only (audio thread never blocked):
    void postMutation(std::function<void(Project&)> fn) {
        std::lock_guard<std::mutex> lock(mutation_mutex_);
        auto current = current_.load(std::memory_order_acquire);
        auto next = std::make_shared<Project>(*current);
        fn(*next);
        current_.store(next, std::memory_order_release);
    }
    ```
    Remove the `mutable std::mutex mutex_` that was acquired in snapshot().
    The mutex_ in the old implementation caused a user-level lock on the audio thread —
    directly contradicting F1-AC4 "no lock acquisition".

    <!-- audit-added M4: BeatSequencer bounds guards -->
    **BeatSequencer process() bounds guards (MUST-HAVE)**:
    Add all of these checks before accessing indexed elements:
    ```cpp
    if (snap.patterns.empty()) return;
    const auto& pattern = snap.patterns[0];
    if (pattern.layers.empty()) return;
    const auto& layer = pattern.layers[0];
    if (layer.type != LayerType::BEAT) return;
    if (layer.drum_tracks.empty()) return;
    ```
    Without these: loading any .layerz without a BEAT layer crashes the audio thread.

    <!-- audit-added M5: duplicate step events cause double-trigger -->
    **BeatSequencer duplicate step guard (MUST-HAVE)**:
    In the drum_track scan loop, after finding the FIRST matching event, break immediately:
    ```cpp
    for (const auto& event : track.events) {
        if (event.step == beat_step) {
            bank.trigger(...);
            break; // do NOT trigger twice if duplicate events exist
        }
    }
    ```
    Duplicate events are a bug scenario (not user-creatable in F1 UI, but possible via
    malformed .layerz or future import). Triggering twice doubles the volume silently.
