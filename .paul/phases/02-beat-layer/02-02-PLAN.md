<objective>
## Goal
LAYERZ displays a functional BEAT layer strip: 4 drum rows each with 16 step buttons,
level knob, and mute button. Toggling steps and adjusting parameters mutates the
ProjectStore and takes effect on the next beat. A BPM spinner and play button appear
in standalone mode.

## Purpose
Closes F1-AC2 (voice param changes audible), F1-AC3 (mute/solo works), and completes
F1-AC4 (RT safety during user interaction). Delivers the first instrument surface
a user can actually play with — no more grey box.

## Output
- `Source/gui/StepRow.h/.cpp` — 16-button row for one drum track
- `Source/gui/VoiceParamPanel.h/.cpp` — level + mute + param1 controls per voice
- `Source/gui/BeatLayerStrip.h/.cpp` — stacks 4 rows; BEAT layer layout
- `Source/PluginEditor.h/.cpp` — replaces grey box with BeatLayerStrip + transport bar
</objective>

<acceptance_criteria>

## AC-1: Step toggle mutates Project and changes audio (F1-AC1 fully)
```gherkin
Given the BEAT layer strip showing 16 buttons per drum track
When the user clicks a step button that was off
Then the button highlights (active state), a new Event is added to that DrumTrack.events
     via postMutation, and on the next loop the hit sounds at that step
When the user clicks an active button
Then the button dims, the Event is removed via postMutation, and the hit no longer sounds
```

## AC-2: Level knob and mute button take effect (F1-AC2, F1-AC3)
```gherkin
Given all 4 drum voices playing
When the user drags the kick level knob from 1.0 to 0.2
Then the kick volume drops noticeably within the current or next audio block
     and the change is reflected in DrumTrack.level in the Project snapshot

When the user clicks the hat mute button
Then the hat stops sounding immediately at the next event
     and the other 3 voices continue uninterrupted
```

## AC-3: param1 slider changes voice character (F1-AC2 continued)
```gherkin
Given a kick voice playing on every quarter beat
When the user drags the kick's param1 (pitch_decay_ms) slider from 80ms to 400ms
Then the audible kick character changes: noticeably longer pitch sweep
     and DrumTrack.param1 is updated in the Project snapshot
```

## AC-4: RT safety during UI interaction (F1-AC4 complete)
```gherkin
Given LAYERZ playing a BEAT pattern with AudioThreadGuard active
When the user rapidly toggles steps, adjusts level knobs, and mutes/unmutes over 60 seconds
Then no LAYERZ_ASSERT_NOT_AUDIO_THREAD fires (i.e. no postMutation or UI code
     runs on the audio thread — all mutations are GUI-thread-only)
```

    **StepRow.h/.cpp:**
    ```cpp
    class StepRow : public juce::Component {
    public:
        // trackIndex: 0=kick,1=snare,2=hat,3=perc
        StepRow(ProjectStore& store, int trackIndex);
        void paint(juce::Graphics&) override;
        void resized() override;
        void updateFromSnapshot(const Project& snap); // called by BeatLayerStrip on timer
    private:
        ProjectStore& store_;
        int trackIndex_;
        juce::TextButton stepButtons_[16];
    };
    ```

    16 TextButton instances in resized() laid out horizontally, equal width.
    Each button onClick: read current project snapshot; toggle presence of Event(step=i)
    in patterns[0].layers[0].drum_tracks[trackIndex_].events via store_.postMutation().
    Event added: step=i, velocity=0.8f, micro_offset_ticks=0.
    Event removed: erase from events where event.step==i.

    Button visual states:
    - Inactive: dark grey background, no label
    - Active (event exists): BEAT crimson fill (juce::Colour(0xFFC0392B))
    - Step index label: monospaced font, 9pt, bottom-right corner of each button

    `updateFromSnapshot()`: called from parent on a timer (≤30fps); reads events[] and
    updates button toggle states without triggering onClick callbacks.
    Avoid postMutation inside updateFromSnapshot — read-only pass.

    **VoiceParamPanel.h/.cpp:**
    ```cpp
    class VoiceParamPanel : public juce::Component {
    public:
        VoiceParamPanel(ProjectStore& store, int trackIndex);
        void resized() override;
        void updateFromSnapshot(const Project& snap);
    private:
        ProjectStore& store_;
        int trackIndex_;
        juce::Slider levelSlider_;   // 0.0 – 1.0, horizontal
        juce::TextButton muteButton_; // toggle
        juce::Slider param1Slider_;  // 10 – 500 (ms / Hz depending on voice)
        juce::Label  voiceLabel_;    // "KICK" / "SNARE" / "HAT" / "PERC"
    };
    ```

    levelSlider_ onChange: postMutation → drum_tracks[trackIndex_].level = newValue
    muteButton_ onClick:   postMutation → drum_tracks[trackIndex_].mute = !current
    param1Slider_ onChange: postMutation → drum_tracks[trackIndex_].param1 = newValue

    Param1 slider range and label per voice:
    - kick: "DECAY" 20–500ms
    - snare: "SNAP" 30–200ms
    - hat: "DECAY" 10–80ms
    - perc: "FREQ" 60–800Hz

    All onChange callbacks: check that the value actually changed before calling postMutation
    (avoid spam mutations from slider movement noise).

    updateFromSnapshot(): refresh slider/button values from current Project without
    triggering onChange callbacks (use setValueNotifyingHost=false or ScopedValueSetter).
