<objective>
## Goal
The Project model exists in memory, loads and saves `.layerz` JSON with round-trip fidelity, the audio thread emits a metronome click synchronized to host transport (plugin) or internal clock (standalone), and profile caps from the F0.99 spike are wired in.

## Purpose
This plan closes F0 by delivering the two load-bearing pieces all future phases build on: the canonical `Project` model (the shared truth between GUI, audio, and LLM bridge) and the audio thread clock/sync harness. Without these, F1 (BEAT layer) has nowhere to read patterns from and no timing reference. The thread-safety pattern from F0.99b is implemented here — it must be chosen before this plan runs.

## Output
- `Source/model/Schema.h` — C++ structs: Project, Pattern, Layer, Event, Aesthetics, Voice
- `Source/model/Project.h/.cpp` — JSON I/O via nlohmann/json; thread-safe access per F0.99b decision
- `Source/audio/ProfileConfig.h` — caps from F0.99 spike (PLUGIN / STANDALONE)
- `Source/audio/Clock.h/.cpp` — host PPQ sync + internal clock; emits metronome click
- `Source/PluginProcessor.h/.cpp` — wired to Clock and Project; click sounds on processBlock
</objective>

    - Thread-safe access: implement EXACTLY the pattern chosen in docs/spikes/F0.99b-thread-safety.md
      - If lock-free-snapshot: expose `getSnapshot()` returning `std::shared_ptr<const Project>` (updated atomically)

    `loadFromFile()` MUST return `juce::Result` (or `std::optional<Project>` + out-param error string),

    unrecognised schema_version. Unknown fields in JSON must be silently skipped (not errored) — this
    enables forward compat when a newer .layerz is opened in an older build.

    <!-- audit-added: atomic write prevents corrupt files on crash mid-save -->

    Project::swing_global) MUST use epsilon comparison: `std::abs(a - b) < 1e-6f`.
    Bit-exact float comparison after JSON serialization/deserialization is non-deterministic.

    Round-trip test: `Project a = loadFromFile(f); saveToFile(a, f2); Project b = loadFromFile(f2);` — a == b must hold. Implement operator== for all structs with float epsilon.


    Wraps timing logic for both plugin and standalone modes:
    - `Clock::prepare(double sampleRate, int blockSize)` — called from prepareToPlay; pre-allocates internal `BeatEvent` array large enough for max beats per block
    - `Clock::process(const juce::AudioPlayHead::PositionInfo& pos, int numSamples)` → returns `juce::ArrayRef<BeatEvent>` (view over pre-allocated internal array — NO heap allocation) <!-- audit-added: "returns list" implies std::vector which allocates; must be a non-owning view over pre-allocated buffer from prepare() -->
    - Plugin mode: reads PPQ from PositionInfo when isPlaying; standalone mode: uses internal counter at configured BPM
    - Internal BPM settable via `Clock::setBpm(double)`
    - Quarter-beat accuracy: compute sample offset = round((ppqPosition - floor(ppqPosition)) * samplesPerBeat)
    - No allocations inside process() — pre-allocate beat event buffer in prepare()

    Click implementation for F0: when Clock returns a beat event, PluginProcessor writes a 1ms Hann-windowed sine burst (440 Hz) at the returned sample offset into the output buffer. <!-- audit-added: rectangular window produces audible click artifact at attack/release due to waveform discontinuity — this would be mistaken for a timing error during AC-2/AC-3 verification --> Apply Hann window: `sample[i] *= 0.5f * (1.0f - std::cos(2.0f * M_PI * i / (burst_samples - 1)))`. This proves timing cleanly.
