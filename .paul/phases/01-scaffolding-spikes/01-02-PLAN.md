      - `NEEDS_MIDI_INPUT TRUE`

    - `target_sources(LAYERZ PRIVATE Source/PluginProcessor.cpp Source/PluginEditor.cpp)`

    Avoid Projucer-style `juceaide` helper scripts.

<task type="auto">
  <name>Task 3: Write GitHub Actions CI workflow</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    Write `.github/workflows/ci.yml`:

    ```yaml
    name: CI
    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    jobs:
      build:
        name: Build (${{ matrix.os }})
        runs-on: ${{ matrix.os }}
        strategy:
          fail-fast: false
          matrix:
            os: [ubuntu-latest, macos-latest, windows-latest]

        steps:
          - uses: actions/checkout@v4

          - name: Cache FetchContent dependencies  # <!-- audit-added: JUCE is ~1.5GB; without cache, CI exceeds the 20-min AC-2 target on first run -->
            uses: actions/cache@v4
            with:
              path: build/_deps
              key: ${{ runner.os }}-fetchcontent-${{ hashFiles('CMakeLists.txt') }}
              restore-keys: ${{ runner.os }}-fetchcontent-

          - name: Install Linux deps
            if: runner.os == 'Linux'
            run: |
              sudo apt-get update
              sudo apt-get install -y \
                libasound2-dev libx11-dev libxrandr-dev libxinerama-dev \
                libxcursor-dev libfreetype6-dev libgl1-mesa-dev liblv2-dev \
                pkg-config

          - name: Configure CMake
            run: cmake -B build -DCMAKE_BUILD_TYPE=Release

          - name: Build
            run: cmake --build build --config Release --parallel

          - name: Upload VST3 artifact
            uses: actions/upload-artifact@v4
            with:
              name: LAYERZ-VST3-${{ matrix.os }}
              path: build/**/*.vst3
    ```

    Notes:
    - `fail-fast: false` so all three platform results are visible on first failure
    - Linux apt deps: the minimal set for JUCE headless build (X11/ALSA/Freetype); may need adjustment once JUCE FetchContent identifies actual link deps
    - No code signing in v0 — CI just verifies the build; distribution signing is F7
    - FetchContent will clone JUCE on first run; cache with `actions/cache` if CI time becomes a problem
  </action>
  <verify>
    Push to main branch; all three matrix jobs pass in GitHub Actions within 20 minutes
  </verify>
